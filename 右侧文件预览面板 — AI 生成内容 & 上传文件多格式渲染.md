
> ⚠️ **架构预审待确认** — 右侧预览面板为 Octo 第三栏新模块，涉及面板宽度状态管理、多文件切换、与对话流联动，需 @Brooks 架构预审后方可推研发。

## 1.需求背景

AI 生成内容（Markdown / PDF / HTML / 代码）和用户上传文件目前在对话流中只以链接或纯文本呈现，阅读体验差。右侧预览面板作为 Octo 第三栏，对文件内容做富文本渲染，用户无需离开对话即可完整阅读和操作。

## 2. 需求概述

| 模块           | 功能点                                                                                               |
| ---------------- | ------------------------------------------------------------------------------------------------------ |
| 对话流文件卡片 | 1. 整卡片可点击打开预览 2. 三态视觉反馈（默认/hover/激活）                                           |
| 预览面板       | 1. 打开、关闭、折叠、拖拽调整宽度 2. 多文件切换（对话内文件下拉） 3. 按文件类型显示工具栏操作项      |
| Markdown 渲染  | 1. GFM 预览（表格、任务列表、代码高亮、数学公式） 2. 预览/源码切换 3. 目录（TOC）按条件出现          |
| HTML 渲染      | 1. 默认视图预览 2. 预览/源码切换 3. 沙箱 iframe 隔离 4. 错误处理                                     |
| PDF 渲染       | 1. 左侧辅助面板（缩略图 / 书签目录） 2. 页码跳转 3. 缩放（适应宽度 / 缩小 / 放大） 4. 键盘与滚轮翻页 |

## 3. 页面结构

### 3.1 对话流文件卡片

1. 卡片内容：文件类型图标、文件名、大小、格式标签
2. 点击整卡片打开右侧预览面板，详见 3.2
3. 三种状态：a. 默认、b. Hover、c. 激活（当前正在预览）
4. 关闭面板时，所有激活态卡片恢复默认态

### 3.2 预览面板 · 基础行为

1. 首次打开初始宽度 400px；用户拖拽左边缘可调整宽度
2. 可调范围 280px – 900px；拖拽过程中禁用过渡动画，保证跟手
3. 折叠：工具栏末尾「✕」按钮 → 面板收起；再次点击任一文件卡片重新打开
4. 一次只渲染一个文件；切换文件时直接替换内容

### 3.3 多文件切换

1. 面板顶部左侧显示「📁 对话内文件 ▾」下拉按钮
2. 点击按钮 → 展开下拉列表，按对话顺序列出所有可预览文件
3. 列表项：文件图标 + 来源标记（`✨` AI 生成 / `📎` 用户上传）+ 文件名
4. 当前预览文件在列表中高亮（品牌色淡底 + 文字加深）
5. 点击列表项 → 切换面板内容，面板不关闭
6. 再次点击按钮 / 点击外部区域 → 收起下拉

### 3.4 工具栏

按文件类型动态装配，分为左侧导航区、中间专属工具区、右侧通用操作区。

**左侧导航区**

1. 所有类型：「📁 对话内文件 ▾」下拉按钮
2. PDF 追加：侧栏切换按钮 ｜ 页码输入 `[n] / [总页数]`
3. Markdown / HTML 追加：`[预览 ｜ 源码]` segmented control

**中间专属工具区**

1. PDF：`适应宽度` ｜ `缩小 −` ｜ `放大 +`
2. 其他类型：无

​**右侧通用操作区**​（从左到右固定顺序）

| 按钮       | 显示规则                              | 交互                                  |
| ------------ | --------------------------------------- | --------------------------------------- |
| 目录       | Markdown 预览模式 且 二级标题 ≥ 3 个 | 点击切换面板左侧目录显隐              |
| 全屏       | 所有类型                              | 点击 → Modal 铺满屏幕；点击close退出 |
| 新标签打开 | 所有类型                              | 在浏览器新 Tab 全屏查看               |
| 回复       | 所有类型                              | 点击 → 文件作为引用填入对话输入框    |
| 下载       | 所有类型                              | 下载原始文件                          |
| ✕         | 所有类型，最右                        | 收起面板                              |

### 3.5 文件类型 · Markdown

1. 预览模式（默认）：渲染 GFM，支持表格、任务列表、代码块语法高亮、图片、链接
2. 数学公式：行内 `$...$` 与块级 `<equation>...</equation>`，KaTeX 渲染
3. 源码模式：原始 markdown 文本，Shiki 按 markdown 语法高亮
4. 视图切换：工具栏「预览 ｜ 源码」segmented control 一键切换
5. 视图记忆：不跨文件，每次打开新文件默认回到预览模式
6. 目录：a. 仅预览模式下出现；源码模式自动隐藏 TOC 按钮b. 出现条件：文档内二级标题（`##`）≥ 3 个c. 点击按钮 → 面板左侧展开目录侧栏，支持 h2 / h3 层级d. 点击目录项 → 滚动定位到对应章节

### 3.6 文件类型 · HTML

**默认视图**

默认为预览，预览和源码可互相切换

**视图切换**

1. 工具栏「预览 ｜ 源码」segmented control
2. 源码模式：Shiki HTML 语法高亮 + 行号 + 等宽字体
3. 视图记忆：不跨文件，每次打开按来源回到默认视图

​**沙箱策略**​（所有 HTML 一致）

1. `<iframe sandbox="allow-same-origin allow-scripts">`
2. 禁止访问父页面 DOM
3. 禁止携带父站点 cookie 跨域请求
4. 不开放 `allow-top-navigation`，防止 iframe 内跳转劫持父窗

### 3.7 文件类型 · PDF

**左侧辅助面板**

1. 默认展开，可通过工具栏「侧栏切换」按钮收起/展开
2. 宽度固定 148px，内容独立滚动
3. 顶部双 Tab：a. 缩略图（始终存在）b. 书签目录（仅当 PDF 自带书签时出现；无书签则只有缩略图 Tab，"书签目录"置灰）
4. 缩略图 Tab：a. 显示全部页面缩略图b. 当前页缩略图以品牌色边框高亮，页码文字也高亮c. 点击任一缩略图 → 主区跳转到对应页
5. 书签目录 Tab：树形结构展示 PDF 书签，点击跳转

**页码跳转**

1. 工具栏显示 `[输入框] / [总页数]`，如 `1 / 12`
2. 输入页码 + Enter → 跳转
3. 非法输入（非数字 / 超出范围）→ 不跳转 → 输入框恢复当前页码
4. 键盘 PageUp / PageDown → 向上/向下翻页
5. 鼠标滚轮 → 连续滚动

**缩放**

1. 三个按钮：`适应宽度`、`缩小 −`、`放大 +`
2. 档位：50% / 75% / 100% / 125% / 150% / 200%
3. 点击「缩小」「放大」在档位间切换
4. 点击「适应宽度」→ 重置为按面板当前宽度撑满
5. 面板打开时默认执行「适应宽度」

### 3.8 文件类型 · 代码 / JSON / CSV / 纯文本

1. 代码（`.py` `.js` `.ts` `.go` `.sh` 等）：Shiki 按对应语言语法高亮 + 行号
2. JSON：a. 树形结构展示，键、字符串、数字、布尔、null 差异化着色b. 节点前「▸」图标，点击展开/收起子节点c. 默认展开第一层
3. CSV：a. 自动解析为表格b. 表头固定，内容区独立滚动c. 行数 > 1000 时仅渲染前 1000 行，顶部提示「显示前 1000 行，完整数据请下载」
4. 纯文本（`.txt` `.log`）：等宽字体展示，保留原始换行

## 4. 异常与边界情况

### 4.1 HTML 渲染报错

1. 触发条件：iframe 渲染抛出 JS 错误或 DOM 构建失败
2. 系统行为：a. 自动将视图切回源码b. 源码区顶部展示红色错误提示条，含错误信息、行号

### 4.2 PDF 解析失败

1. 触发条件：PDF 文件损坏 / 密码加密 / 格式不支持
2. 系统行为：面板中央展示占位图 + 文案「文件解析失败，可下载到本地查看」+ 「下载」按钮

### 4.3 文件过大

1. 触发条件：文件 > 20 MB
2. 系统行为：不加载渲染，面板中央展示占位图 + 文案「文件过大（{实际大小}），建议下载到本地查看」+ 「下载」按钮

### 4.4 不支持的文件类型

1. 触发条件：扩展名不在支持列表内（详见 5.1）
2. 系统行为：面板中央展示占位图 + 文件名 + 文案「该文件类型暂不支持预览」+ 「下载」按钮
3. 不报错、不 Toast

### 4.5 下载失败

1. 触发条件：文件已过期 / 网络异常 / 权限不足
2. 系统行为：Toast「下载失败，请稍后重试」

### 4.6 CSV 行数超限

1. 触发条件：行数 > 1000
2. 系统行为：仅渲染前 1000 行，表格上方提示「显示前 1000 行，完整数据请下载」

### 4.7 Markdown 源码模式下的  目录

1. 触发条件：在源码模式下
2. 系统行为：目录按钮自动隐藏；若目录侧栏已打开，同时关闭

### 4.8 切换文件期间的加载状态

1. 从点击文件到内容渲染完成之间，预览内容区显示骨架屏
2. 加载 > 3 秒，显示 spinner + 文案「加载中…」
3. 加载失败：回退到文件名 + 「重试」按钮

## 7. 验收标准

1. 点击对话流中任意文件卡片，右侧预览面板在 1 秒内打开并展示骨架屏，最终渲染对应内容。
2. 面板可通过 ✕ 按钮关闭；关闭后再次点击文件卡片可重新打开。
3. 面板宽度可通过拖拽左边缘调整，范围限制在 280px–900px 之间，拖拽过程无卡顿。
4. 「对话内文件」下拉列表可展开，显示当前对话所有可预览文件，切换后面板内容更新，不关闭面板。
5. Markdown 文件：默认显示 GFM 渲染，支持预览/源码切换；文档含 ≥3 个二级标题时工具栏出现「目录」按钮，点击展开目录侧栏。
6. HTML 文件：默认显示预览（沙箱 iframe），可切换源码；渲染出错时自动切回源码并显示红色错误提示条。
7. PDF 文件：左侧辅助面板默认展开，显示缩略图；含书签时出现书签目录 Tab；页码输入框支持跳转，缩放按档位切换；键盘 PageUp/PageDown 可翻页。
8. 代码文件：Shiki 语法高亮 + 行号；JSON 显示可折叠树形结构；CSV 以表格呈现，行数 > 1000 时只渲染前 1000 行并顶部提示。
9. 文件 > 20MB 时不渲染，面板中央显示文案「文件过大（{实际大小}），建议下载到本地查看」及「下载」按钮。
10. 文件类型不在支持列表内时，面板显示「该文件类型暂不支持预览」及「下载」按钮，不报错不 Toast。
11. 下载失败时显示 Toast「下载失败，请稍后重试」。
12. 全屏按钮点击后面板以 Modal 形式铺满屏幕，点击 close 退出。
13. 「回复」按钮点击后，文件作为引用填入对话输入框。

## 5. 本期不做（明确排除）

### 5.1 不支持的文件类型

1. `.docx` / `.xlsx` / `.pptx`：不支持
2. 图片、视频、音频：对话流内已渲染，不进入面板

### 5.2 不支持的能力

1. 文件编辑：所有预览只读
2. 代码执行 / 运行：HTML 沙箱不开放 JS 运行以外的环境
3. 分享 / 发布外链：排除
4. PDF 翻译全文：排除
5. PDF 截图提问：排除
6. PDF 图表视图提取：排除

以上能力 Claude Artifacts / 豆包有部分提供，本期排除，后续视用户反馈迭代。
